# Entender cómo contribuir con informes de errores y PR

Esta página explica cómo abrir la guía de contribución incluida en AetherSDR. La guía cubre cómo presentar informes de errores y enviar pull requests para el proyecto.

## Antes de comenzar

- No se requiere conexión a una radio para abrir los temas de ayuda.

## Pasos

1. Haga clic en `Help` en la barra de menú.
2. Haga clic en `Contributing to AetherSDR...`.
3. Se abre el HelpDialog y muestra la guía de contribución en el visor de Markdown.
4. Lea el contenido. Los enlaces externos del documento se abren en su navegador predeterminado.
5. Haga clic en `Close` cuando termine.

## Qué hace cada control

| Control | Descripción |
|---|---|
| `AETHERSDR OFFLINE HELP` | Encabezado de marca que se muestra sobre el título del tema. |
| Title | Muestra el nombre del tema de ayuda abierto. |
| Subtitle | Muestra una descripción de una línea: "Bundled help is available even when your station computer is offline." |
| Markdown viewer | Renderiza la guía de contribución. Los enlaces son clicables y se abren de forma externa. |
| Hint / footer | Muestra: "Tip: The Help menu keeps each guide separate so you can reopen just the topic you need." |
| `Close` | Cierra el diálogo. |

## Consejos

- Cada tema de ayuda en el menú `Help` abre su propio diálogo independiente, por lo que puede volver a abrir únicamente este tema sin navegar por otras guías.
- Si la guía de contribución no se carga correctamente, AetherSDR muestra un mensaje de respaldo en el visor de Markdown que indica que el archivo de ayuda no está disponible. En ese caso, reinstale la aplicación o reporte el recurso de ayuda faltante.
