# Usar AGC-T en RX para suprimir el ruido de banda por debajo de un nivel mínimo elegido

El applet Aetherial AGC-T (RX) aplica un expansor descendente del lado del cliente al audio recibido. Úselo para reducir el ruido de banda persistente, QRM o silbidos entre señales, atenuando el audio que cae por debajo de un nivel que usted defina.

## Antes de comenzar

- El applet Aetherial AGC-T (RX) permanece oculto hasta que la etapa Gate esté habilitada en el lado RX. Habilítela primero desde el widget CHAIN o desde el editor flotante Gate para el lado RX.
- El applet se encuentra dentro del contenedor principal Aetherial Audio (TXDSP) en el Applet Panel. Abra el Applet Panel si no está visible: `View > Applet Panel`.

## Pasos

1. En el Applet Panel, ubique el subcontenedor **Aetherial AGC-T** dentro del contenedor principal Aetherial Audio (TXDSP).
2. Habilite la etapa Gate para el lado RX usando el widget CHAIN (haga clic simple en la etapa Gate) o haga doble clic en la etapa Gate del widget CHAIN para abrir el editor flotante **Aetherial Gate — RX**.
3. Observe la **curva de transferencia** (Transfer curve) — la bola en movimiento sigue el nivel de audio entrante en tiempo real. Note la posición de la bola durante el ruido de banda y durante una señal.
4. Gire el mando **Thresh** para establecer el nivel por debajo del cual el gate comienza a atenuar. Ajústelo justo por encima del piso de ruido y por debajo de la señal más débil que desee escuchar. Valor predeterminado: -40.0 dB; rango: -80.0 a 0.0 dB. Este valor se guarda como `ClientGateRxThresholdDb`.
5. Gire el mando **Ratio** para controlar con qué intensidad se recorta el audio por debajo del umbral. A 2.0:1 (predeterminado) el gate actúa como un expansor suave; a 10.0:1 el corte es muy pronunciado. Rango: 1.0 a 10.0. Se guarda como `ClientGateRxRatio`.
6. Gire el mando **Return** para establecer la banda muerta de histéresis. El gate se abre cuando la entrada supera Thresh y no vuelve a cerrarse hasta que la entrada cae por debajo de Thresh menos el valor de Return. Esto evita el traqueteo rápido cerca del umbral. Valor predeterminado: 2.0 dB; rango: 0.0 a 20.0 dB. Se guarda como `ClientGateRxReturnDb`. Una banda vertical de color cian suave aparece en la **curva de transferencia** entre Thresh − Return y Thresh para hacer visible esta zona de retención.
7. Gire el mando **Floor** para establecer la atenuación máxima que el gate puede aplicar. El valor predeterminado de -15.0 dB deja un residuo de ruido tenue en lugar de cortar completamente al silencio. Rango: -80.0 a 0.0 dB. Se guarda como `ClientGateRxFloorDb`.
8. Gire el mando **Release** para controlar con qué rapidez se cierra el gate después de que la entrada cae por debajo de Thresh menos el valor de Return. Valor predeterminado: 100 ms; rango: 5 a 2000 ms. Se guarda como `ClientGateRxReleaseMs`. Un Release más largo suena más natural; uno más corto cierra con mayor precisión entre señales.
9. Observe la **barra de reducción de ganancia** (Gain-reduction bar) mientras hay ruido de banda sin señal presente. La franja ámbar se rellena desde la derecha a medida que se aplica reducción de ganancia. La marca en -15 dB indica la posición predeterminada de Floor. Ajuste Thresh y Floor hasta que la barra permanezca rellena durante el ruido y se vacíe rápidamente cuando aparezca una señal.

## Consejos

- Comience con Thresh en -40.0 dB y auméntelo lentamente hasta que la barra de reducción de ganancia se rellene de forma sólida durante el ruido de banda, pero se abra con claridad cuando aparezca una señal.
- Ajuste Floor a -15.0 dB inicialmente. Redúzcalo (más negativo) solo si el ruido residual sigue siendo molesto. Evite valores de Floor muy bajos, ya que pueden hacer que el audio suene antinatural entre señales.
- Use el mando **Return** para eliminar el traqueteo del gate en señales que rondan cerca del umbral. Aumente Return hasta que el gate permanezca abierto de forma continua durante las pausas breves y, a continuación, verifique que aún se cierra durante los silencios más prolongados.
- La banda de cian suave en la **curva de transferencia** proporciona retroalimentación visual inmediata al ajustar Return — una banda más ancha indica una banda muerta de histéresis mayor.
- Los cambios realizados en el editor flotante **Aetherial Gate — RX** y en el mosaico del applet se mantienen sincronizados — el mosaico consulta el motor aproximadamente cada 33 ms y actualiza sus mandos para reflejar los cambios.
- El AGC-T opera completamente del lado del cliente y no requiere conexión con el radio para configurarse.

## Resolución de problemas

- **El applet Aetherial AGC-T no es visible en el Applet Panel** — La etapa Gate en el lado RX no ha sido habilitada. Habilítela mediante el widget CHAIN para el lado RX. El applet permanece oculto hasta que la etapa esté activa.
- **La barra de reducción de ganancia no muestra actividad incluso durante el ruido de banda** — Thresh está ajustado demasiado bajo. El piso de ruido no alcanza el umbral. Aumente Thresh hacia 0 dB hasta que la barra muestre relleno ámbar durante el ruido.
- **Las señales se están cortando junto con el ruido** — Thresh está ajustado demasiado alto. Baje Thresh de modo que solo el piso de ruido active la atenuación y las señales deseadas abran el gate.
- **El gate suena abrupto o entrecortado entre señales** — Aumente el valor de Release para permitir que el gate se cierre de forma más gradual después de cada señal.
- **El gate traquetea rápidamente cerca del umbral** — Aumente el valor de Return para ampliar la banda muerta de histéresis de modo que el gate no vuelva a abrirse y cerrarse en los bordes de señal próximos a Thresh.
