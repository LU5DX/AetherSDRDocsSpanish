# Uso de AGC-T en RX para suprimir el ruido de banda por debajo de un piso seleccionado

El applet Aetherial AGC-T (RX) aplica un expansor descendente del lado del cliente al audio recibido. Úselo para reducir el ruido de banda persistente, la QRM o el siseo entre señales, atenuando el audio que cae por debajo de un nivel que usted establece.

## Antes de comenzar

- El applet Aetherial AGC-T (RX) está oculto hasta que la etapa Gate esté habilitada en el lado RX. Actívela primero mediante el widget CHAIN o el editor flotante Gate para el lado RX.
- El applet se encuentra dentro del contenedor principal Aetherial Audio (TXDSP) en el Panel de Applets. Abra el Panel de Applets si no está visible: `View > Applet Panel`.

## Pasos

1. En el Panel de Applets, localice el subcontenedor **Aetherial AGC-T** dentro del contenedor principal Aetherial Audio (TXDSP).
2. Habilite la etapa Gate para el lado RX usando el widget CHAIN (un solo clic en la etapa Gate) o haga doble clic en la etapa Gate en el widget CHAIN para abrir el editor flotante **Aetherial Gate — RX**.
3. Observe la **Curva de transferencia**: la bola en vivo se mueve con su nivel de audio entrante. Note dónde se ubica la bola durante el ruido de banda versus durante una señal.
4. Gire la perilla **Thresh** para establecer el nivel por debajo del cual la puerta comienza a atenuar. Ajústelo justo por encima del piso de ruido y por debajo de la señal más débil que desee escuchar. Valor predeterminado: -40.0 dB; rango: -80.0 a 0.0 dB. Este valor se guarda como `ClientGateRxThresholdDb`.
5. Gire la perilla **Ratio** para controlar qué tan agresivamente se corta el audio por debajo del umbral. En 2.0:1 (predeterminado) la puerta actúa como un expansor suave; en 10.0:1 corta de forma muy fuerte. Rango: 1.0 a 10.0. Se guarda como `ClientGateRxRatio`.
6. Gire la perilla **Return** para establecer la banda muerta de histéresis. La puerta se abre cuando la entrada supera Thresh y no se cierra de nuevo hasta que la entrada cae por debajo de Thresh menos el valor de Return. Esto evita el parloteo rápido cerca del umbral. Valor predeterminado: 2.0 dB; rango: 0.0 a 20.0 dB. Se guarda como `ClientGateRxReturnDb`. Aparece una banda vertical cian suave en la **Curva de transferencia** entre Thresh − Return y Thresh para hacer visible esta zona adherente.
7. Gire la perilla **Floor** para establecer la atenuación máxima que la puerta puede aplicar. El valor predeterminado de -15.0 dB deja un ruido residual leve en lugar de cortar hasta el silencio. Rango: -80.0 a 0.0 dB. Se guarda como `ClientGateRxFloorDb`.
8. Gire la perilla **Release** para controlar qué tan rápido se cierra la puerta después de que la entrada cae por debajo de Thresh menos el valor de Return. Valor predeterminado: 100 ms; rango: 5 a 2000 ms. Se guarda como `ClientGateRxReleaseMs`. Una liberación más larga suena más natural; una liberación más corta cierra con mayor firmeza entre señales.
9. Observe la **Barra de reducción de ganancia** mientras haya ruido de banda presente sin señal. La franja ámbar se llena desde la derecha a medida que se aplica la reducción de ganancia. La marca en -15 dB indica la posición predeterminada de Floor. Ajuste Thresh y Floor hasta que la barra permanezca llena durante el ruido y se vacíe rápidamente cuando aparezca una señal.
10. Cuando la etapa Gate está desviada, todo el mosaico del applet se atenúa aproximadamente al 55 % de opacidad. Esta señal visual confirma que la etapa está inactiva; los valores de las perillas se conservan y vuelven a tener efecto tan pronto como reactive la etapa.

## Consejos

- Comience con Thresh en -40.0 dB y súbalo lentamente hasta que la barra de reducción de ganancia se llene sólidamente durante el ruido de banda, pero se abra limpiamente cuando aparezca una señal.
- Ajuste Floor a -15.0 dB inicialmente. Redúzcalo (más negativo) solo si el ruido residual sigue siendo molesto. Evite valores de Floor muy bajos, ya que pueden hacer que el audio suene poco natural entre señales.
- Use la perilla **Return** para eliminar el parloteo de la puerta en señales que rondan cerca del umbral. Aumente Return hasta que la puerta se mantenga abierta limpiamente durante las pausas breves, luego verifique que aún se cierre durante silencios más largos.
- La banda cian suave en la **Curva de transferencia** proporciona retroalimentación visual inmediata mientras ajusta Return: una banda más ancha significa una banda muerta de histéresis más grande.
- Los cambios realizados en el editor flotante **Aetherial Gate — RX** y en el mosaico del applet se mantienen sincronizados; el mosaico consulta el motor aproximadamente cada 33 ms y actualiza sus perillas para coincidir.
- El AGC-T funciona completamente del lado del cliente y no requiere conexión al radio para configurarse.
- Cuando la etapa Gate está desviada, el mosaico del applet se renderiza con opacidad reducida (aproximadamente 55 %). Esto coincide con el comportamiento de atenuación de la curva EQ cuando su etapa está desviada y facilita ver de un vistazo qué etapas de procesamiento están activas.

## Solución de problemas

- **El applet Aetherial AGC-T no está visible en el Panel de Applets** — La etapa Gate en el lado RX no se ha habilitado. Actívela mediante el widget CHAIN para el lado RX. El applet está oculto hasta que la etapa esté activa.
- **La barra de reducción de ganancia no muestra actividad incluso durante el ruido de banda** — Thresh está demasiado bajo. El piso de ruido no está alcanzando el umbral. Suba Thresh hacia 0 dB hasta que la barra muestre un relleno ámbar durante el ruido.
- **Las señales se están cortando junto con el ruido** — Thresh está demasiado alto. Baje Thresh para que solo el piso de ruido active la atenuación y las señales deseadas abran la puerta.
- **La puerta suena abrupta o entrecortada entre señales** — Aumente el valor de Release para permitir que la puerta se cierre más gradualmente después de cada señal.
- **La puerta parlotea rápidamente cerca del umbral** — Aumente el valor de Return para ampliar la banda muerta de histéresis, de modo que la puerta no se reabra y se vuelva a cerrar en los bordes de la señal cerca de Thresh.
- **El mosaico del applet aparece atenuado y la puerta no procesa audio** — La etapa Gate está desviada. El mosaico con opacidad reducida es intencional. Vuelva a habilitar la etapa mediante el widget CHAIN para restaurar el procesamiento completo y la opacidad total.
