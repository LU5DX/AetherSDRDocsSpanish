# Configurar el retardo de break-in en CW

El retardo de break-in en CW controla cuánto tiempo espera el radio después de la última pulsación de tecla antes de volver a recepción. Ajustar esto evita una conmutación QSK entrecortada mientras permite un rápido retorno a RX entre palabras o caracteres.

## Antes de comenzar

- AetherSDR debe estar conectado al radio. El applet Phone/CW muestra controles solo cuando hay una conexión activa con el radio.
- El slice activo debe estar en un modo CW. El subpanel CW reemplaza automáticamente al subpanel Phone cuando se selecciona CW en el slice activo.

## Pasos

1. Localice el botón de bandeja **P/CW** en la barra lateral derecha y confirme que el applet sea visible. Si el applet no es visible, haga clic en el botón de bandeja **P/CW** para mostrarlo.
2. Confirme que se muestre el subpanel CW. Si en lugar de eso se muestran los controles Phone, cambie el slice activo a un modo CW usando el selector de modo en el área principal de VFO.
3. Localice el control deslizante **Delay (CW)** en el subpanel CW.
4. Arrastre el control deslizante **Delay (CW)** hacia la izquierda para disminuir el retardo o hacia la derecha para aumentarlo. El valor se aplica al radio de inmediato.
5. Alternativamente, haga clic en la visualización del valor junto al control deslizante y escriba un número directamente (0-2000 ms). Presione Enter para aplicar el valor.

## Consejos

- Un retardo de 0 ms con **Breakin** habilitado proporciona operación QSK completa. Aumente el retardo para reducir el desgaste del relé durante el envío rápido.
- El control deslizante **Delay (CW)** avanza en incrementos de 10 ms. Para un ajuste preciso, haga clic en la pista del control deslizante y use las teclas de flecha (si los atajos de teclado están habilitados en `View > Keyboard Shortcuts`).
- La visualización del valor es un QLineEdit editable. Haga clic en él para escribir un valor preciso y luego presione Enter. El control deslizante se moverá para coincidir.

## Notas para v26.5.1

- **Reemplazo del medidor SW ALC:** Ambos indicadores ALC (panel Phone y panel CW) ahora leen desde MeterModel::swAlcChanged, que proporciona el pico SSB post-ALC de software en dBFS. Esto reemplaza la ruta HWALC anterior (voltaje RCA) que producía lecturas sin sentido. Los nuevos indicadores usan un rango de -20 a 0 dBFS con visualización de llenado desde la derecha y un umbral rojo en -3 dBFS.
- **Reflejo ALC de panel dual:** El panel Phone ahora tiene su propio indicador ALC (m_alcGaugePhone) que refleja el indicador ALC del panel CW (m_alcGaugeCw). Ambos indicadores reciben los mismos datos desde la única ranura updateAlc() conectada a MeterModel::swAlcChanged, asegurando lecturas consistentes entre modos de voz y CW.

## Notas para v0.9.8

- **Campos de valor editables:** Las cuatro etiquetas de valor CW (Delay, Speed, Sidetone Volume, Pitch) ahora son widgets QLineEdit con QIntValidator. Haga clic en cualquier valor y escriba un número directamente para una entrada precisa.
- **Corrección del almacenamiento en caché del valor de retardo:** La función setCwDelay ahora almacena en caché el valor de inmediato, evitando que la emisión del radio devuelva el control deslizante al valor anterior.

## Notas para v0.9.7

- **Comportamiento de Breakin corregido:** La alternancia **Breakin** ahora controla completamente si las rutas de tecleo CW del teclado y MIDI activan TX automáticamente. Anteriormente, una envolvente interna de auto-PTT anulaba el estado **Breakin OFF** y suprimía el tiempo de retención QSK. Esa envolvente ha sido eliminada. Con **Breakin** desactivado, las teclas ponen caracteres en cola y PTT debe activarse manualmente; con **Breakin** activado, los bordes de tecla activan TX inmediatamente y `break_in_delay` gobierna el tiempo de retención del relé.
- **Indicador de compresión cerrado en estado de transmisión:** El indicador **Compression** en el subpanel Phone ahora lee 0 dB durante la recepción. Solo muestra un valor distinto de cero cuando el enclavamiento del radio informa TRANSMITTING y el procesador de voz está habilitado. Esto evita que aparezcan lecturas obsoletas de la cadena de TX mientras está escuchando.
- **Control deslizante de ganancia de micrófono en modo RADE:** Cuando el modo RADE está activo, el control deslizante **Mic gain** actúa como un control de ganancia RADE del lado del cliente y usa la configuración `PcMicGain`, la misma clave utilizada para la fuente de micrófono PC. El valor del control deslizante ya no se envía al radio como `mic_level` mientras RADE está activo, lo que de otro modo sobrescribiría silenciosamente la configuración de micrófono del hardware. Cuando se desactiva el modo RADE, el control deslizante vuelve a reflejar el nivel de micrófono informado por el radio.
- **Indicador de nivel en modo RADE:** Cuando el modo RADE está activo, el indicador **Level** permanece activo durante la recepción (no requiere que `met_in_rx` esté configurado), de forma coherente con el comportamiento existente de la fuente de micrófono PC. Cuando se desactiva el modo RADE, el indicador se restablece a -150 dBFS.

## Notas para v0.9.3

- **Indicador de nivel (panel Phone):** Cuando la fuente de micrófono está configurada en **PC**, el indicador Level ahora aparece inmediatamente al conectar sin esperar a que se configure el indicador `met_in_rx` del radio. Esto se debe a que la medición del micrófono PC se ejecuta del lado del cliente y es independiente de `met_in_rx`. Para todas las demás fuentes de micrófono, el comportamiento de supresión anterior no cambia: el indicador lee -150 dBFS cuando `met_in_rx` está desactivado y el radio no está transmitiendo.
- **Actualización de VOX / panel Phone:** Los configuradores VOX ahora emiten `phoneStateChanged`, por lo que el subpanel Phone se actualiza instantáneamente cuando se activa o desactiva VOX mediante un atajo de teclado. No se necesita interacción manual con el panel para ver el estado actual de VOX.
- **Tono lateral en Windows:** El flujo de tono lateral CW (CwSidetoneGenerator de baja latencia) ahora se inicia inmediatamente al conectar en Windows. Si anteriormente tenía que desactivar y volver a activar **Sidetone** después de conectar para escuchar el tono lateral local, esta solución alternativa ya no es necesaria.
